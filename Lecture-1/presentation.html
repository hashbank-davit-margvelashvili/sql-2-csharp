The Big Picture

.NET 10 — Why This Version?

.NET 8Nov 2023LTS (3 years)Active
.NET 9Nov 2024STS (18 months)Active
.NET 10Nov 2025LTS (3 years)We use this

Why LTS? In banking, stability matters. LTS versions receive security patches for 3 years. .NET 10 ships with C# 14, the latest language version.

.NET 10 is to C# what SQL Server 2022 is to T-SQL — the runtime determines which language features are available.

Key .NET Components

CLR

Common Language Runtime — the engine that runs your code. Handles memory, type safety, exceptions.

BCL

Base Class Library — pre-built code for I/O, networking, collections, JSON, crypto. Like built-in system procedures.

SDK

Software Development Kit — the compiler, dotnet CLI, project templates, NuGet package manager.

Runtime

The minimal components to run an app (CLR + BCL). In production, you only need this — not the full SDK.

Managed Code

C# code on the CLR is called managed code because the runtime manages it:

  • Memory is automatically allocated and freed (garbage collection)
  • Array bounds are checked at runtime (no buffer overflows)
  • Type safety is enforced at compile time
  • Exceptions are structured and propagated safely
In SQL Server, you never manually manage memory — the engine handles it. .NET works the same way. You focus on business logic, not memory allocation.

.NET vs .NET Framework: When someone says ".NET" without a version, they mean modern .NET (5+). ".NET Framework" (1.0–4.8) is the legacy Windows-only version. We use modern .NET 10.

Program.cs — Your Entry Point

C# (Top-level statements)

// Program.cs - the simplest C# program
Console.WriteLine("Hello, Banking World!");

One line. That's the whole program. The compiler wraps it in a class and Main method automatically.

T-SQL equivalent

-- The T-SQL equivalent
PRINT 'Hello, Banking World!'

SQL Server lets you run statements without wrapping them in a procedure. C# does the same with top-level statements.

Behind the scenes, the compiler generates:

internal class Program {
    static void Main(string[] args) {
        Console.WriteLine("Hello, Banking World!");
    }
}

.gitignore — What NOT to Track

Never commit these:

  • bin/ — compiled output
  • obj/ — intermediate build files
  • .vs/ — Visual Studio settings
  • *.user — personal preferences
  • Passwords and secrets

Always commit these:

  • *.cs — source code
  • *.csproj — project configuration
  • *.sln — solution file
  • .gitignore — the ignore rules
  • appsettings.json — non-secret config

Tip: Run dotnet new gitignore to generate a comprehensive .NET-specific gitignore file automatically.

Knowledge Check

Q: What is the C# equivalent of the SQL Server Query Optimizer?

A) The C# Compiler
B) The JIT (Just-In-Time) Compiler
C) NuGet
D) Visual Studio
Correct! The JIT compiler optimizes IL code into native machine code at runtime, just like the Query Optimizer generates optimal execution plans for T-SQL.
Not quite. The C# compiler converts C# to IL (like parsing T-SQL). The JIT compiler then optimizes IL into machine code at runtime — that's the equivalent of the Query Optimizer.

Knowledge Check

Q: Which command creates a new console project and runs it?

A) dotnet create console && dotnet execute
B) dotnet new console -n MyApp && cd MyApp && dotnet run
C) dotnet build console -n MyApp && dotnet start
D) csc MyApp.cs && MyApp.exe
Correct! dotnet new console -n MyApp creates the project, then dotnet run compiles and executes it in one step.
Not quite. The correct commands are dotnet new (to create) and dotnet run (to build + execute). Option D would work with the old .NET Framework compiler, but we use the modern dotnet CLI.

HSP Project — The Journey Begins

Your capstone project: extract a real HSP stored procedure into a .NET microservice.

HSP (Today)

  • All logic in stored procedures
  • Tightly coupled
  • Hard to unit test
  • Manual deployments
  • Limited monitoring

Microservice (Session 20)

  • C# services with clear separation
  • Loosely coupled, independent
  • Comprehensive automated tests
  • Automated CI/CD pipeline
  • Structured logging & health checks

Dual value: You learn .NET development AND produce a real microservice that moves the bank toward its target architecture.

Activity: Identify Your Candidate

Choose an HSP stored procedure to extract. Good candidates are:

✓

Clear, bounded responsibility — does one thing well

✓

Self-contained — doesn't call 50 other procedures

✓

Well-defined inputs/outputs — you know exactly what goes in and comes out

✓

Represents a business capability — could stand on its own as a service

Document your choice: procedure name, input parameters, output structure, business rules, error handling, dependent tables, calling applications.
