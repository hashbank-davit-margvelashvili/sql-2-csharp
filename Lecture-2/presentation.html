Decimal vs Double — Critical for Banking

Never use float or double for money. They have floating-point precision errors.

Precision comparison
// WRONG -- floating point error
double wrong = 0.1 + 0.2;
Console.WriteLine($"double: {wrong}");

// CORRECT -- exact decimal arithmetic
decimal correct = 0.1m + 0.2m;
Console.WriteLine($"decimal: {correct}");
double: 0.30000000000000004 decimal: 0.3
In SQL Server, you use DECIMAL for money, not FLOAT. Same rule in C# — always use decimal with the m suffix.
decimal balance = 45_000.50m;   // The 'm' means "decimal literal"
decimal fee = 2.5m;             // Underscores for readability

Strings — Simpler Than SQL

SQL Server

  • CHAR(n) / VARCHAR(n)
  • NCHAR(n) / NVARCHAR(n)
  • Fixed vs variable length
  • ASCII vs Unicode
  • Must declare size: (50), (MAX)

C#

  • Just string
  • Always Unicode
  • Always variable length
  • No size limit declaration
  • Immutable (modifications create new strings)
string name = "Davit";              // Like NVARCHAR -- always Unicode
string empty = "";                    // Empty string (not NULL)
string? nullable = null;             // Nullable string (? = can be NULL)
string long = "Any length, no MAX needed";

Value Types vs Reference Types

A concept SQL doesn't have. This is fundamental to understanding C#.

Value Type — IS the data

Variable contains the actual value. Copy = independent clone.

int a = 42;
int b = a;     // Copy the value
b = 100;       // Only b changes

Console.WriteLine(a); // 42
Console.WriteLine(b); // 100
a
42
b
100

Reference Type — Points to data

Variable holds a reference. Copy = both point to same data.

int[] a = { 1, 2, 3 };
int[] b = a;   // Copy the reference
b[0] = 999;   // Both are affected!

Console.WriteLine(a[0]); // 999!
Console.WriteLine(b[0]); // 999

The var Keyword

Let the compiler figure out the type. Still strictly typed — just shorthand.

Type inference
var count = 42;                // Compiler: this is int
var name = "Davit";             // Compiler: this is string
var balance = 15000.50m;        // Compiler: this is decimal
var today = DateTime.Now;        // Compiler: this is DateTime

Console.WriteLine(count.GetType());
Console.WriteLine(name.GetType());
Console.WriteLine(balance.GetType());
Console.WriteLine(today.GetType());
System.Int32 System.String System.Decimal System.DateTime

var is NOT dynamic typing. The type is locked at compile time. var x = 42; x = "hello"; is a compile error.

Null Operators — Your New Superpowers

?? Null-Coalescing

// C#
string? name = null;
string display = name ?? "Unknown";
// display = "Unknown"
-- T-SQL equivalent
SELECT ISNULL(@name, 'Unknown')
SELECT COALESCE(@name, 'Unknown')

??= Null-Coalescing Assignment

string? name = null;
name ??= "Default";
// Sets name to "Default" ONLY if null

?. Null-Conditional

string? name = null;

// WITHOUT ?. -- CRASH!
// int len = name.Length;

// WITH ?. -- safe, returns null
int? len = name?.Length;  // null

// Combine with ?? for default
int safe = name?.Length ?? 0;
// safe = 0

Chaining

string city = customer?.Address?.City
               ?? "N/A";

Knowledge Check

Q: What is the output of this code?

string? city = null;
int length = city?.Length ?? -1;
Console.WriteLine(length);
A) 0
B) -1
C) Throws NullReferenceException
D) Compile error
Correct! city?.Length returns null because city is null (safe navigation). Then ?? -1 provides the default value -1.
Not quite. city?.Length safely returns null (no crash). Then ?? -1 kicks in, giving us -1.

Knowledge Check

Q: Which C# type should you use for a bank account balance?

A) double — it has more range
B) float — it uses less memory
C) decimal — exact precision for financial data
D) int — store as minor units (cents)
Correct! decimal provides exact arithmetic for financial calculations. double and float have floating-point precision errors (0.1 + 0.2 ≠ 0.3).
decimal is the correct answer. double and float have floating-point errors: 0.1 + 0.2 = 0.30000000000000004. Always use decimal for money.

Knowledge Check

Q: What happens here?

int a = 10;
int b = a;
b = 99;
Console.WriteLine(a);
A) Prints 10 — int is a value type, b got a copy
B) Prints 99 — b is a reference to a
C) Compile error — can't reassign b
D) Prints 0 — a was reset
Correct! int is a value type. b = a copies the value 10 into a separate box. Changing b doesn't affect a.
int is a value type — b = a copies the value, not a reference. a remains 10. This would be different with reference types (arrays, classes).
